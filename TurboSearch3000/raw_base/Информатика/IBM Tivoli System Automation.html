Введение

Семейство программных продуктов IBM Tivoli® System Automation предназначено для обеспечения высокой доступности (high availability), а также, автоматизации IT ресурсов в компьютерных кластерах (cluster). Под IT ресурсами понимаются программные приложения и их компоненты, процессы, IP инфраструктура, внешняя память и.т.д. Под кластером понимается совокупность компьютеров с разделяемыми ресурсами. Область применения System Automation от одного компьютера до IT инфраструктуры целого предприятия.

Функциональность

Технология System Automation различает два аспекта: Автоматизация (Automation) и высокая доступность (high availability). Автоматизация подразумевает запуск и остановку ресурсов согласно заданной политике (policy) с учетом зависимостей (dependencies) ресурсов друг от друга.

Например, заданная цель для ресурса А - быть активным (online). Ресурс А имеет зависимость от ресурса B:

   А --> StartAfter --> B

то есть, для запуска ресурса А необходимо, чтобы ресурс B был активным. System Automation обеспечивает запуск ресурса B и, когда ресурс B станет активным, запустит ресурс А.

Такие зависимости между ресурсами или группами ресурсов могут быть следующих видов:

   Порядок запуска или остановки:
       - StartAfter
       - StopAfter
       - ForceDown

   Локальность или нелокальность, т.е. требование к ресурсам работать на одном узле (операционной системе) или на разных:
       - Collocated
       - AntiCollocated

   Группирование ресурсов и вложенность групп.

Все ресурсы, группы и зависимости описываются в специальной таблице (или графе), называемой policy. Кроме того, в policy задается желаемое состояние ресурсов и их групп (например, Online). Когда policy активируется, System Automation пытается наиболее полно удовлетворить заданную в policy цель.

Такая технология называется goal driven Automation - целевая Автоматизация.

Стандартные политики

Чтобы облегчить пользователю написание политик для конкретных индустриальных решений, System Automation предлагает шаблоны политик, которые остается лишь приспособить к данной конфигурации. Такие шаблоны (precanned policies) поставляются для следующих продуктов и решений (смотри иллюстрацию). Например, политика SAP описывает взаимозависимость компонент в среде SAP, a System Automation обеспечивает желаемое состояние компонент, запуская и останавливая их в заданном порядке, а также поддерживая их высокую доступность.

System Automation Application Manager

Описанное выше решение позволяет автоматизировать и делать высоко доступными ресурсы в пределах гомогенных кластеров. Это означает, что все узлы кластера должны иметь одинаковую архитектуру и операционную систему, например Linux на Intel x86. Однако вычислительные центры современных предприятий оснащены самыми различными платформами, на которых работают приложения, требующие высокой доступности. Приложения на различных платформах, а значит и в разных кластерах, могут зависеть друг от друга (crosscluster dependency). Для автоматизации таких мультиплаттформных (мультикластерных) сред используется System Automation Application Manager (SAAM), позволяющий создавать policies с учетом мультикластерных зависимостей.

Графический интерфейс пользователя

Базированный на технологии WebSphere Графический интерфейс пользователя (GUI) позволяет в удобной форме отслеживать состояние ресурсов, узлов и кластеров. Кроме того, пользователь имеет возможность запускать и останавливать ресурсы, а также временно отключать Автоматизацию для того или иного ресурса или для целых групп (requests).

Поддерживаемые платформы и операционные системы

IBM Tivoli® System Automation поддерживает следующие платформы:

  • AIX
  • Linux
  • SUN Solaris
  • MS Windows Server 2003/2008
  • z/OS
