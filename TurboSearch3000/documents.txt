				<div id="mw-content-text" lang="ru" dir="ltr" class="mw-content-ltr"><p><b>Единый указатель ресурсов</b> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a>&#160;<i><span lang="en" xml:lang="en"><b>URL — Uniform Resource Locator</b></span></i>)&#160;— единообразный локатор (определитель местонахождения) ресурса. По-<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">английски</a> «URL» целиком произносится как <span class="IPA"><a href="/wiki/%D0%9C%D0%B5%D0%B6%D0%B4%D1%83%D0%BD%D0%B0%D1%80%D0%BE%D0%B4%D0%BD%D1%8B%D0%B9_%D1%84%D0%BE%D0%BD%D0%B5%D1%82%D0%B8%D1%87%D0%B5%D1%81%D0%BA%D0%B8%D0%B9_%D0%B0%D0%BB%D1%84%D0%B0%D0%B2%D0%B8%D1%82" title="Международный фонетический алфавит">/ɜː(ɹ)l/</a></span>, по-русски чаще говорят [<i>у-эр-э́л</i>], [<i>ю-ар-эл</i>] или [<i>урл</i>] (<i><a href="/wiki/%D0%A1%D0%BB%D0%B5%D0%BD%D0%B3" title="Сленг">сленг</a></i>). Ранее назывался <b>Universal Resource Locator</b>&#160;— универсальный локатор ресурса. URL&#160;— это стандартизированный способ записи адреса ресурса в сети <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BD%D0%B5%D1%82" title="Интернет">Интернет</a>.</p>

<h2><span class="editsection">[<a href="/w/index.php?title=URL&amp;action=edit&amp;section=1" title="Редактировать раздел «История»">править</a>]</span> <span class="mw-headline" id=".D0.98.D1.81.D1.82.D0.BE.D1.80.D0.B8.D1.8F">История</span></h2>
<p>URL был изобретён <a href="/wiki/%D0%91%D0%B5%D1%80%D0%BD%D0%B5%D1%80%D1%81-%D0%9B%D0%B8,_%D0%A2%D0%B8%D0%BC" title="Бернерс-Ли, Тим">Тимом Бернерсом-Ли</a> в <a href="/wiki/1990_%D0%B3%D0%BE%D0%B4" title="1990 год">1990&#160;году</a> в стенах Европейского совета по ядерным исследованиям (<a href="/wiki/%D0%A4%D1%80%D0%B0%D0%BD%D1%86%D1%83%D0%B7%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Французский язык">фр.</a>&#160;<i><span lang="fr" xml:lang="fr">Conseil Européen pour la Recherche Nucléaire, <a href="/wiki/CERN" title="CERN" class="mw-redirect">CERN</a></span></i>) в <a href="/wiki/%D0%96%D0%B5%D0%BD%D0%B5%D0%B2%D0%B0" title="Женева">Женеве</a>, <a href="/wiki/%D0%A8%D0%B2%D0%B5%D0%B9%D1%86%D0%B0%D1%80%D0%B8%D1%8F" title="Швейцария">Швейцария</a>. URL стал фундаментальной инновацией в Интернете. Изначально URL предназначался для обозначения мест расположения ресурсов (чаще всего файлов) во <a href="/wiki/%D0%92%D1%81%D0%B5%D0%BC%D0%B8%D1%80%D0%BD%D0%B0%D1%8F_%D0%BF%D0%B0%D1%83%D1%82%D0%B8%D0%BD%D0%B0" title="Всемирная паутина">Всемирной паутине</a>. Сейчас URL применяется для обозначения адресов почти всех ресурсов Интернета. Стандарт URL закреплён в документе <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc1738">RFC 1738</a>, прежняя версия была определена в <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc1630">RFC 1630</a>. Сейчас URL позиционируется как часть более общей системы идентификации ресурсов <a href="/wiki/URI" title="URI">URI</a>, сам термин URL постепенно уступает место более широкому термину <b>URI</b>. Стандарт URL регулируется организацией <a href="/wiki/IETF" title="IETF" class="mw-redirect">IETF</a> и её подразделениями.</p>
<h2><span class="editsection">[<a href="/w/index.php?title=URL&amp;action=edit&amp;section=2" title="Редактировать раздел «Структура URL»">править</a>]</span> <span class="mw-headline" id=".D0.A1.D1.82.D1.80.D1.83.D0.BA.D1.82.D1.83.D1.80.D0.B0_URL">Структура URL</span></h2>
<p>Изначально локатор URL был разработан как система для максимально естественного указания на местонахождения ресурсов в сети. Локатор должен был быть легко расширяемым и использовать лишь ограниченный набор <a href="/wiki/ASCII" title="ASCII">ASCII</a>‐символов (к примеру, <a href="/wiki/%D0%9F%D1%80%D0%BE%D0%B1%D0%B5%D0%BB" title="Пробел">пробел</a> никогда не применяется в URL). В связи с этим, возникла следующая традиционная форма записи URL:</p>
<pre>
&lt;схема&gt;://&lt;логин&gt;:&lt;пароль&gt;@&lt;хост&gt;:&lt;порт&gt;/&lt;URL‐путь&gt;?&lt;параметры&gt;#&lt;якорь&gt;
</pre>
<p>В этой записи:</p>
<dl>
<dt><i>схема</i>&#160;</dt>
<dd>схема обращения к ресурсу; в большинстве случаев имеется в виду <a href="/wiki/%D0%A1%D0%B5%D1%82%D0%B5%D0%B2%D1%8B%D0%B5_%D0%BF%D1%80%D0%BE%D1%82%D0%BE%D0%BA%D0%BE%D0%BB%D1%8B" title="Сетевые протоколы" class="mw-redirect">сетевой протокол</a></dd>
<dt><i>логин</i>&#160;</dt>
<dd>имя пользователя, используемое для доступа к ресурсу</dd>
<dt><i>пароль</i>&#160;</dt>
<dd><a href="/wiki/%D0%9F%D0%B0%D1%80%D0%BE%D0%BB%D1%8C" title="Пароль">пароль</a> указанного пользователя</dd>
<dt><i>хост</i>&#160;</dt>
<dd>полностью прописанное <a href="/wiki/%D0%94%D0%BE%D0%BC%D0%B5%D0%BD%D0%BD%D0%BE%D0%B5_%D0%B8%D0%BC%D1%8F" title="Доменное имя">доменное имя</a> <a href="/wiki/%D0%A5%D0%BE%D1%81%D1%82" title="Хост">хоста</a> в системе <a href="/wiki/DNS" title="DNS">DNS</a> или <a href="/wiki/IP-%D0%B0%D0%B4%D1%80%D0%B5%D1%81" title="IP-адрес">IP-адрес</a> хоста в форме четырёх групп <a href="/wiki/%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D1%81%D1%87%D0%B8%D1%81%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F" title="Система счисления">десятичных</a> <a href="/wiki/%D0%A7%D0%B8%D1%81%D0%BB%D0%BE" title="Число">чисел</a>, разделённых точками; числа&#160;— целые в интервале от 0 до 255.</dd>
<dt><i>порт</i>&#160;</dt>
<dd><a href="/wiki/%D0%9F%D0%BE%D1%80%D1%82_(TCP/UDP)" title="Порт (TCP/UDP)" class="mw-redirect">порт</a> хоста для подключения</dd>
<dt><i>URL-путь</i>&#160;</dt>
<dd>уточняющая <a href="/wiki/%D0%98%D0%BD%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D1%86%D0%B8%D1%8F" title="Информация">информация</a> о месте нахождения ресурса; зависит от протокола.</dd>
<dt><i>параметры</i>&#160;</dt>
<dd>строка запроса с передаваемыми на сервер (<a href="/wiki/HTTP#GET" title="HTTP">методом GET</a>) параметрами. Разделитель параметров&#160;— знак <b>&amp;</b>.</dd>
</dl>
<p>Пример:&#160;?параметр_1=значение_1&amp;параметр_2=значение_2&amp;параметр3=значение_3</p>
<dl>
<dt><i>якорь</i>&#160;</dt>
<dd>идентификатор «якоря», ссылающегося на некоторую часть (раздел) открываемого документа. С развитием <a href="/wiki/AJAX" title="AJAX">AJAX</a> в этой части стали размещать параметры для JavaScript на странице, и страница в зависимости от указанного якоря может в браузере выглядеть по-разному. Для обеспечения возможности индексирования подобных страниц поисковыми системами компания Google предложила подход, предполагающий использование знака&#160;! в начале якоря и некоторую поддержку на сервере сайта<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup>.</dd>
</dl>
<p>На сегодняшний день <a href="/wiki/%D0%A2%D0%B8%D0%BC_%D0%91%D0%B5%D1%80%D0%BD%D0%B5%D1%81-%D0%9B%D0%B8" title="Тим Бернес-Ли" class="mw-redirect">Тим Бернес-Ли</a> признаёт, что символ двойной косой черты в структуре URL является избыточным<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup>.</p>
<h2><span class="editsection">[<a href="/w/index.php?title=URL&amp;action=edit&amp;section=3" title="Редактировать раздел «Схемы (протоколы) URL»">править</a>]</span> <span class="mw-headline" id=".D0.A1.D1.85.D0.B5.D0.BC.D1.8B_.28.D0.BF.D1.80.D0.BE.D1.82.D0.BE.D0.BA.D0.BE.D0.BB.D1.8B.29_URL">Схемы (протоколы) URL</span></h2>
<p>Общепринятые схемы (протоколы) URL включают:</p>
<ul>
<li>ftp&#160;— Протокол передачи файлов <a href="/wiki/FTP" title="FTP">FTP</a></li>
<li>http&#160;— Протокол передачи <a href="/wiki/%D0%93%D0%B8%D0%BF%D0%B5%D1%80%D1%82%D0%B5%D0%BA%D1%81%D1%82" title="Гипертекст">гипертекста</a> <a href="/wiki/HTTP" title="HTTP">HTTP</a></li>
<li>rtmp&#160;— Real Time Messaging Protocol проприетарный протокол потоковой передачи данных, в основном используется для передачи потокового видео и аудиопотоков с веб-камер через интернет.</li>
<li>rtsp&#160;— Потоковый протокол реального времени.</li>
<li>https&#160;— Специальная реализация протокола <a href="/wiki/HTTP" title="HTTP">HTTP</a>, использующая шифрование (как правило, <a href="/wiki/SSL" title="SSL">SSL</a> или <a href="/wiki/TLS" title="TLS">TLS</a>)</li>
<li>gopher&#160;— Протокол <a href="/wiki/Gopher_(%D1%81%D0%B5%D1%82%D0%B5%D0%B2%D0%BE%D0%B9_%D0%BF%D1%80%D0%BE%D1%82%D0%BE%D0%BA%D0%BE%D0%BB)" title="Gopher (сетевой протокол)" class="mw-redirect">Gopher</a></li>
<li>mailto&#160;— Адрес <a href="/wiki/%D0%AD%D0%BB%D0%B5%D0%BA%D1%82%D1%80%D0%BE%D0%BD%D0%BD%D0%B0%D1%8F_%D0%BF%D0%BE%D1%87%D1%82%D0%B0" title="Электронная почта">электронной почты</a></li>
<li>news&#160;— Новости <a href="/wiki/Usenet" title="Usenet">Usenet</a></li>
<li>nntp&#160;— Новости <a href="/wiki/Usenet" title="Usenet">Usenet</a> через протокол <a href="/wiki/NNTP" title="NNTP">NNTP</a></li>
<li>irc&#160;— Протокол <a href="/wiki/IRC" title="IRC">IRC</a></li>
<li>prospero&#160;— Служба каталогов <i><a href="/w/index.php?title=Prospero&amp;action=edit&amp;redlink=1" class="new" title="Prospero (страница отсутствует)">Prospero</a> Directory Service</i></li>
<li>telnet&#160;— Ссылка на интерактивную сессию <a href="/wiki/Telnet" title="Telnet">Telnet</a></li>
<li>wais&#160;— База данных системы <a href="/wiki/WAIS" title="WAIS">WAIS</a></li>
<li>xmpp&#160;— Протокол XMPP (часть <a href="/wiki/Jabber" title="Jabber" class="mw-redirect">Jabber</a>)</li>
<li>file&#160;— Имя локального <a href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB" title="Файл">файла</a></li>
<li>data&#160;— Непосредственные данные (<a href="/wiki/Data:_URL" title="Data: URL">Data: URL</a>)</li>
<li>tel&#160;— звонок по указаному телефону</li>
</ul>

<li>ed2k&#160;— <a href="/wiki/%D0%A4%D0%B0%D0%B9%D0%BB%D0%BE%D0%BE%D0%B1%D0%BC%D0%B5%D0%BD%D0%BD%D0%B0%D1%8F_%D1%81%D0%B5%D1%82%D1%8C" title="Файлообменная сеть">Файлообменная сеть</a> <a href="/wiki/EDonkey2000_(%D1%81%D0%B5%D1%82%D1%8C)" title="EDonkey2000 (сеть)">eDonkey</a>, построенная по принципу <a href="/wiki/%D0%9E%D0%B4%D0%BD%D0%BE%D1%80%D0%B0%D0%BD%D0%B3%D0%BE%D0%B2%D0%B0%D1%8F_%D1%81%D0%B5%D1%82%D1%8C" title="Одноранговая сеть">P2P</a></li>

<h2><span class="editsection">[<a href="/w/index.php?title=URL&amp;action=edit&amp;section=4" title="Редактировать раздел «Кодирование URL»">править</a>]</span> <span class="mw-headline" id=".D0.9A.D0.BE.D0.B4.D0.B8.D1.80.D0.BE.D0.B2.D0.B0.D0.BD.D0.B8.D0.B5_URL">Кодирование URL</span></h2>
<p>Появление адресов URL стало существенным нововведением в <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BD%D0%B5%D1%82" title="Интернет">Интернете</a>. Однако с момента его изобретения и по сей день стандарт URL обладает серьёзным недостатком&#160;— в нём можно использовать только ограниченный набор <a href="/wiki/%D0%9F%D0%B5%D1%87%D0%B0%D1%82%D0%BD%D1%8B%D0%B5_%D1%81%D0%B8%D0%BC%D0%B2%D0%BE%D0%BB%D1%8B" title="Печатные символы">символов</a>, даже меньший, нежели <span style="white-space: nowrap;">в <a href="/wiki/ASCII" title="ASCII">ASCII</a>:</span> латинские буквы, цифры и лишь некоторые знаки препинания. Если мы захотим использовать в URL символы <a href="/wiki/%D0%9A%D0%B8%D1%80%D0%B8%D0%BB%D0%BB%D0%B8%D1%86%D0%B0" title="Кириллица">кириллицы</a>, или <a href="/wiki/%D0%98%D0%B5%D1%80%D0%BE%D0%B3%D0%BB%D0%B8%D1%84" title="Иероглиф">иероглифы</a>, или, скажем, специфические символы <a href="/wiki/%D0%A4%D1%80%D0%B0%D0%BD%D1%86%D1%83%D0%B7%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Французский язык">французского языка</a>, то нужные нам символы должны быть перекодированы особым образом.</p>
<p>В русскоязычной <a href="/wiki/%D0%92%D0%B8%D0%BA%D0%B8%D0%BF%D0%B5%D0%B4%D0%B8%D1%8F" title="Википедия">Википедии</a> ежедневно приходится видеть пример кодирования URL, поскольку <a href="/wiki/%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Русский язык">русский язык</a> использует символы кириллицы. Например, строка вида:</p>
<pre>
http://ru.wikipedia.org/wiki/Микрокредит
</pre>
<p>кодируется в URL как:</p>
<pre>
http://ru.wikipedia.org/wiki/%D0%9C%D0%B8%D0%BA%D1%80%D0%BE%D0%BA%D1%80%D0%B5%D0%B4%D0%B8%D1%82
</pre>
<p>Такое преобразование происходит в два этапа: сначала каждый символ кириллицы кодируется в Юникоде (<a href="/wiki/UTF-8" title="UTF-8">UTF-8</a>) в последовательность из двух байтов, а затем каждый байт этой последовательности записывается в <a href="/wiki/%D0%A8%D0%B5%D1%81%D1%82%D0%BD%D0%B0%D0%B4%D1%86%D0%B0%D1%82%D0%B5%D1%80%D0%B8%D1%87%D0%BD%D0%B0%D1%8F_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D1%81%D1%87%D0%B8%D1%81%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F" title="Шестнадцатеричная система счисления">шестнадцатеричном</a> представлении:</p>
<pre>
М → D0 и 9C → %D0%9C
и → D0 и B8 → %D0%B8
к → D0 и BA → %D0%BA
р → D1 и 80 → %D1%80, и т. д.
</pre>
<p>Перед каждым таким шестнадцатеричным кодом байта, согласно спецификации URL<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup>, ставится <a href="/wiki/%D0%97%D0%BD%D0%B0%D0%BA_%D0%BF%D1%80%D0%BE%D1%86%D0%B5%D0%BD%D1%82%D0%B0" title="Знак процента">знак процента</a> (%)&#160;— отсюда даже возник английский термин «percent‐encoding», обозначающий способ кодирования символов в URL и URI.</p>

<p>Поскольку такому преобразованию подвергаются буквы всех алфавитов, кроме базовой <a href="/wiki/%D0%9B%D0%B0%D1%82%D0%B8%D0%BD%D0%B8%D1%86%D0%B0" title="Латиница" class="mw-redirect">латиницы</a>, то URL со словами на подавляющем большинстве языков (кроме <a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">английского</a>, <a href="/wiki/%D0%98%D1%82%D0%B0%D0%BB%D1%8C%D1%8F%D0%BD%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Итальянский язык">итальянского</a>, <a href="/wiki/%D0%9B%D0%B0%D1%82%D0%B8%D0%BD%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Латинский язык">латинского</a>) может утратить способность восприниматься людьми.</p>
<p>Это всё входит в противоречие с принципом <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BD%D0%B0%D1%86%D0%B8%D0%BE%D0%BD%D0%B0%D0%BB%D0%B8%D0%B7%D0%BC_(%D0%BF%D0%BE%D0%BB%D0%B8%D1%82%D0%B8%D0%BA%D0%B0)" title="Интернационализм (политика)" class="mw-redirect">интернационализма</a>, провозглашаемого всеми ведущими организациями <a href="/wiki/%D0%98%D0%BD%D1%82%D0%B5%D1%80%D0%BD%D0%B5%D1%82" title="Интернет">Интернета</a>, включая <a href="/wiki/W3C" title="W3C" class="mw-redirect">W3C</a> и <a href="/wiki/ISOC" title="ISOC" class="mw-redirect">ISOC</a>. Эту проблему призван решить стандарт <a href="/wiki/IRI" title="IRI">IRI</a> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a>&#160;<i><span lang="en" xml:lang="en">International Resource Identifier</span></i>)&#160;— международных идентификаторов ресурсов, в которых можно было бы без проблем использовать символы Юникода, и которые поэтому не ущемляли бы права других <a href="/wiki/%D0%AF%D0%B7%D1%8B%D0%BA_(%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D0%B7%D0%BD%D0%B0%D0%BA%D0%BE%D0%B2)" title="Язык (система знаков)" class="mw-redirect">языков</a>. Хотя заранее сложно сказать, смогут ли когда‐либо идентификаторы IRI заменить столь широко используемые URL (и <a href="/wiki/URI" title="URI">URI</a> в целом).</p>
<h2><span class="editsection">[<a href="/w/index.php?title=URL&amp;action=edit&amp;section=5" title="Редактировать раздел «Инициатива PURL»">править</a>]</span> <span class="mw-headline" id=".D0.98.D0.BD.D0.B8.D1.86.D0.B8.D0.B0.D1.82.D0.B8.D0.B2.D0.B0_PURL">Инициатива PURL</span></h2>
<p>Ещё один кардинальный недостаток URL состоит в отсутствии гибкости. Ресурсы во Всемирной паутине и Интернете перемещаются, а <a href="/wiki/%D0%93%D0%B8%D0%BF%D0%B5%D1%80%D1%81%D1%81%D1%8B%D0%BB%D0%BA%D0%B0" title="Гиперссылка">ссылки</a> в виде URL остаются, указывая на уже отсутствующие ресурсы. Это особенно болезненно для электронных библиотек, каталогов и энциклопедий. Для решения этой проблемы были предложены постоянные локаторы <a href="/wiki/PURL" title="PURL">PURL</a> (<a href="/wiki/%D0%90%D0%BD%D0%B3%D0%BB%D0%B8%D0%B9%D1%81%D0%BA%D0%B8%D0%B9_%D1%8F%D0%B7%D1%8B%D0%BA" title="Английский язык">англ.</a>&#160;<i><span lang="en" xml:lang="en">Persistent Uniform Resource Locator</span></i>). В сущности это те же URL, но они указывают не на конкретное место расположения ресурса, а на запись в базе данных PURL, где, в свою очередь, записан уже конкретный URL‐адрес ресурса. При обращении к PURL <a href="/wiki/%D0%A1%D0%B5%D1%80%D0%B2%D0%B5%D1%80_(%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5)" title="Сервер (приложение)" class="mw-redirect">сервер</a> находит нужную запись в этой <a href="/wiki/%D0%91%D0%B0%D0%B7%D0%B0_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85" title="База данных">базе данных</a> и перенаправляет запрос уже на конкретное местоположение ресурса. Если адрес ресурса меняется, то нет нужды исправлять все бесчисленные ссылки на него&#160;— достаточно лишь изменить запись в БД. В настоящий момент эта идея не стандартизирована и не имеет широкого распространения.</p>
